# ATD 图片榜单系统设计文档

## 1. 目标与范围
本系统用于快速创建图片榜单并导出结果图，覆盖以下能力：

- 配置榜单（标题、等级、颜色）
- 解析图片来源（本地文件、图片链接、网页链接）
- 拖拽排序
- 导出榜单主区域图片

不在当前范围：

- 用户系统与多用户协作
- 云端持久化与历史版本管理
- 服务端任务队列与异步渲染

## 2. 总体架构

### 2.1 模块划分
- 前端（Vue）
  - `SetupView`：配置与图片输入
  - `BoardView`：拖拽排序与导出
  - `lib/board.ts`：前端数据模型、约束与 localStorage 存取
- 后端（Spring Boot）
  - `ImageResolveController`：图片解析与代理接口
  - `ImageResolveService`：URL 校验、抓取、解析、缓存、安全过滤

### 2.2 数据流
1. 用户在前端输入或拖拽内容。
2. 若是链接，前端调用 `/api/image/resolve` 获取可用图片地址。
3. 前端展示图片并保存配置到 localStorage。
4. 进入榜单页后，拖拽状态持续保存到 localStorage。
5. 导出时仅截图 `rank-table` 区域，生成 PNG。

## 3. 后端设计

### 3.1 接口
1. `POST /api/image/resolve`
   - 输入：`ResolveImageRequest(url)`
   - 输出：`ResolveImageResponse(status, resolvedImageUrl, proxyUrl, reason)`

2. `GET /api/image/proxy`
   - 输入：`target`（目标图片 URL）
   - 输出：图片字节流与 `Content-Type`

### 3.2 解析策略
- 直链判断：基于常见图片后缀快速识别。
- 非直链解析：
  - 优先 `og:image` / `twitter:image`
  - 回退到 `img[src|data-src|data-original]`
  - 使用尺寸评分（宽 * 高）选择候选图

### 3.3 安全策略
- 仅允许 `http/https`
- 禁止内网、回环、本地链路等地址（防 SSRF）
- 响应体大小限制：
  - HTML 最大 2MB
  - 图片最大 10MB
- 请求超时与短期缓存（5 分钟）控制资源开销

## 4. 前端设计

### 4.1 配置页（SetupView）
- 负责榜单结构编辑与图片输入。
- 支持三种输入模式：
  - 选择文件（本地图片）
  - 粘贴链接文本（自动提取 URL）
  - 拖拽文件/链接/网页内容（自动解析）
- 解析后给出摘要信息（数量、失败数、尺寸/大小提示）。

### 4.2 榜单页（BoardView）
- 使用 `vuedraggable` 管理拖拽排序。
- 分区：
  - `rank-table`：各等级排序区
  - `pool-panel`：未排序图片区
- 导出逻辑：
  - 等待图片加载完成
  - 使用 `html2canvas` 截图 `rank-table`
  - 输出 PNG 下载文件

### 4.3 本地持久化（board.ts）
- `BOARD_CONFIG_KEY`：保存榜单配置
- `BOARD_STATE_KEY`：保存拖拽状态
- 加载时做结构化校验与兼容性处理，异常则回退默认状态

## 5. 关键数据结构
- `TierConfig`：等级配置（id、label、color）
- `ImageItem`：图片实体（id、src、originalUrl、解析状态）
- `BoardConfig`：整体配置（title、tiers、images）
- `BoardState`：运行时排序状态（rankings、pool）

## 6. 可维护性说明
- 前后端职责清晰：解析与代理在后端，交互与状态在前端。
- 统一错误码映射，前端可直接转换用户提示。
- 注释已补齐核心入口、流程方法与关键策略点，便于二次开发。

## 7. 后续优化建议
1. 引入服务端缓存层（如 Redis）减少重复解析压力。
2. 增加导出模式（`cover` / `contain`）切换。
3. 为图片解析增加重试与可观测指标（成功率、耗时、失败原因分布）。
4. 增加自动化测试（前端单测 + 后端解析集成测试）。
