# ATD 图片榜单系统设计文档（纯前端）

## 1. 目标与范围
系统用于快速创建图片榜单并导出 PNG，包含：

- 配置榜单（标题、等级、颜色）
- 导入图片（本地文件、图片直链 URL）
- 拖拽排序（等级区与待排序池互拖）
- 导出榜单主区域图片

不在范围内：

- 网页链接解析提图
- 后端代理、后端缓存、服务端安全校验
- 多用户与云端持久化

## 2. 总体架构

### 2.1 模块划分
- `SetupView`：榜单配置和图片导入
- `BoardView`：拖拽排序和导出
- `lib/board.ts`：数据模型、约束、`localStorage` 持久化与内存兜底

### 2.2 数据流
1. 用户导入本地图片或输入图片直链 URL。
2. 前端在浏览器内尝试加载图片并提取尺寸信息。
3. 成功图片进入图片池，失败图片以失败态展示。
4. 配置和排序状态实时写入 `localStorage`（失败时使用内存兜底）。
5. 榜单页仅导出 `rank-table` 区域为 PNG。

## 3. 前端关键设计

### 3.1 输入策略
- 本地文件：`FileReader` 读为 `dataURL`。
- URL 输入：只接受 `http/https` 且路径后缀为常见图片扩展名。
- 非直链网页 URL 统一失败提示，不做 HTML 解析。

### 3.2 状态模型
- `BoardConfig`：`title + tiers + images`
- `BoardState`：`rankings + pool`
- `ImageItem`：包含 `resolveStatus/resolveReason/width/height/qualityHint`

### 3.3 导出策略
- 使用 `html2canvas` 截图 `rank-table`。
- 导出前等待图片加载/解码，降低空白图与拉伸风险。
- 保留失败态提示，避免导出流程被单图失败阻断。

## 4. 可维护性说明
- 单仓库、单构建链路（Vite），减少前后端联调成本。
- 数据结构集中在 `board.ts`，便于后续扩展（模板、存档、分享）。
- UI 失败态与提示统一，定位问题更直接。

## 5. 后续优化建议
1. 增加批量导入校验报告（失败原因按类型聚合）。
2. 增加图片裁剪模式切换（`cover/contain`）以适配不同内容。
3. 支持榜单模板预设与导入导出 JSON。
