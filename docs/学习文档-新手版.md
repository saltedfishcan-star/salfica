# ATD 学习文档（新手版）

## 1. 先理解这个项目在做什么

ATD 是一个图片榜单工具，你可以：

1. 添加图片（选文件、粘贴链接、拖拽内容）
2. 设置等级（A/B/C...）和颜色
3. 拖拽图片进行排序
4. 导出榜单区域为 PNG

它由两部分组成：

- 后端（Java Spring Boot）：负责“解析链接”和“代理图片”
- 前端（Vue）：负责界面交互、拖拽和导出

## 2. 目录速览（先看这些）

```text
src/main/java/com/example/atd/

  image/

  views/SetupView.vue                 # 配置页
  views/BoardView.vue                 # 榜单页
  lib/board.ts                        # 数据模型与本地存储
  router/index.ts                     # 页面路由
```

---

## 3. 运行项目（最短路径）

### 后端

在项目根目录：

```bash
mvn spring-boot:run
```

### 前端

在 `vue_atd` 目录：

```bash
npm install
npm run dev
```

## 4. 新手阅读顺序（建议照着走）

1. `vue_atd/src/lib/board.ts`
   - 看数据结构：`BoardConfig`、`BoardState`、`ImageItem`
   - 看 localStorage 如何保存和恢复

2. `vue_atd/src/views/SetupView.vue`
   - 看图片输入流程：文件、链接、拖拽
   - 看点击“进入榜单”前做了哪些校验

3. `vue_atd/src/views/BoardView.vue`
   - 看拖拽排序是如何绑定数据的
   - 看导出 PNG 的流程

4. `src/main/java/com/example/atd/image/ImageResolveService.java`
   - 看后端如何把网页链接解析成图片链接
   - 看它如何做安全校验（防内网地址）

## 5. 关键知识点（够用版）

### 5.1 Vue 组合式 API

- `ref()`：声明状态
- `watch()`：监听状态变化（例如自动保存）
- `<script setup>`：更简洁的单文件组件写法

### 5.2 拖拽库 vuedraggable

- 通过 `v-model` 直接绑定数组
- 拖动后数组自动变化，页面自动刷新

### 5.3 localStorage 持久化

- 配置与排序状态都在浏览器本地保存
- 刷新页面后仍可恢复

### 5.4 后端链接解析

- 直链图片：直接可用
- 网页链接：抓取 HTML 后提取 `og:image` 或 `img`
- 不安全地址：直接拒绝（SSRF 防护）

## 6. 常见修改入口

1. 想改页面样式：`vue_atd/src/views/*.vue` 和 `vue_atd/src/assets/main.css`
2. 想改图片解析规则：`ImageResolveService.java`
3. 想改导出行为：`BoardView.vue` 中 `downloadRankTable()`
4. 想改数据字段：`board.ts` + 前后端 DTO

## 7. 新手练习建议

3. 给解析失败列表增加“重试”按钮

做完这 3 个练习，你就基本掌握这个项目了。
