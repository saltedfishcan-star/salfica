# ATD 项目学习文档（技术与组件构建全景）

## 1. 项目定位与总体架构

ATD 是一个“图片榜单（Tier List）”应用，核心能力是：

- 输入图片（本地文件、图片链接、网页链接）
- 配置榜单等级与颜色
- 拖拽排序
- 导出榜单主区域图片（PNG）

项目采用前后端分离：

- 后端：Spring Boot（`src/main/java/...`）
- 前端：Vue 3 + TypeScript + Vite（`vue_atd/`）

运行时数据流：

1. 前端输入链接或文件。
2. 对链接，前端调用后端解析接口 `/api/image/resolve`。
3. 后端返回可用图片地址与代理地址。
4. 前端将配置/状态存到 `localStorage`。
5. 榜单页拖拽排序后，导出 `rank-table` 为 PNG。

---

## 2. 后端技术与知识点

### 2.1 Spring Boot Web 基础

应用入口是 `AtdApplication`，使用 `@SpringBootApplication` 启动容器。  
控制层使用 `@RestController` 暴露 JSON 接口与二进制代理接口。

关键知识点：

- `@PostMapping` / `@GetMapping`：REST 接口定义
- `ResponseEntity<byte[]>`：返回图片二进制流
- `@ExceptionHandler`：集中处理业务异常

### 2.2 图片解析与代理（核心服务）

`ImageResolveService` 是后端核心，职责包括：

- URL 解析与校验（仅 http/https）
- SSRF 风险防护（过滤内网/回环/链路本地地址）
- 远程抓取（`HttpClient` + 超时 + 最大响应体限制）
- HTML 图片提取（`Jsoup`）
- 缓存解析结果（ConcurrentHashMap + TTL）
- 构造代理 URL 供前端展示

关键知识点：

- `java.net.http.HttpClient`：JDK 原生 HTTP 客户端
- `Jsoup.parse()`：HTML DOM 解析
- `record`：轻量 DTO（请求/响应）
- `ConcurrentHashMap`：线程安全缓存

### 2.3 HTML 图片提取策略

服务优先从社交元标签提取：

- `og:image`
- `twitter:image`

找不到再回退到 `<img>` 标签候选（`src` / `data-src` / `data-original`），并按尺寸分数选择最优图。

### 2.4 错误处理与可观测性

后端错误会转成可识别 reason，例如：

- `invalid_url`
- `upstream_fetch_failed`
- `response_too_large`
- `blocked_image_host`

前端再将 reason 映射为用户提示。

## 3. 前端技术与知识点

### 3.1 Vue 3 组合式 API

项目主写法是 `<script setup lang="ts">`，核心响应式 API：

- `ref`：状态
- `computed`：派生值
- `watch`：状态持久化（同步到 localStorage）

### 3.2 路由系统

`vue-router` 管理页面：

- `/setup`：配置页
- `/board`：榜单页
- `/about`：懒加载示例页

知识点：

- `createWebHistory`：H5 History 路由
- route-level code splitting：按路由拆包

### 3.3 拖拽能力

`vuedraggable` 用于图片卡片在不同列表间拖拽：

- 排名区 `rankings[tier.id]`
- 待排序池 `pool`

知识点：

- `v-model` 直接绑定数组
- 组配置 `group` 支持跨列表拖拽
- ghost/chosen class 提供拖拽视觉反馈

### 3.4 导出图片能力

榜单导出基于 `html2canvas`，仅导出 `rank-table`。

关键处理：

- 导出前等待图片解码完成
- 指定导出宽高为 `rankTable.scrollWidth/scrollHeight`
- 克隆 DOM 时将 `<img>` 替换为 `background-size: cover` 块，规避高图压扁问题

### 3.5 本地持久化与数据校验

`vue_atd/src/lib/board.ts` 负责：

- 类型定义：`BoardConfig` / `BoardState` / `ImageItem` / `TierConfig`
- 配置与状态读写：`load/save/reset`
- 结构校验与兼容恢复：非法数据自动回退默认值

这是项目“状态稳定性”的核心。

## 4. 组件构建过程（逐组件）

以下按“从无到有”的构建思路说明每个主要组件如何实现。

### 4.1 后端组件

#### 4.1.1 `AtdApplication`

构建步骤：

1. 新建 Spring Boot 项目。
2. 保留标准 `main` 启动入口。
3. 通过 `spring-boot-maven-plugin` 支持打包与运行。

#### 4.1.2 `ImageResolveController`

构建步骤：

1. 定义统一前缀 `@RequestMapping("/api/image")`。
2. 增加 `POST /resolve`：接受 URL，返回解析结果。
3. 增加 `GET /proxy`：转发并输出图片二进制。
4. 增加 `@ExceptionHandler`：统一错误结构。

#### 4.1.3 `ImageResolveService`

构建步骤：

1. 先做 URL 合法性与公网校验（防 SSRF）。
2. 实现 `resolve()`：缓存 -> 直链判断 -> 抓取与解析。
3. 实现 HTML 解析提图策略（meta 优先，img 回退）。
4. 实现 `proxyImage()`：拉取目标图并返回内容类型。
5. 增加超时、大小限制、缓存 TTL，确保稳定性。

#### 4.1.4 `ResolveImageRequest/Response`、`ProxyImagePayload`、`ImageProxyException`

构建步骤：

1. 用 `record` 定义输入/输出载体，减少样板代码。
2. `ResolveImageResponse` 提供 `ok/failed` 工厂方法。
3. 自定义异常封装状态码与错误原因，供控制层统一输出。

### 4.2 前端组件

#### 4.2.1 `main.ts` + `App.vue`

构建步骤：

1. 初始化 Vue 应用。
2. 挂载路由。
3. 在 `App.vue` 放置壳层 `<RouterView />`。

#### 4.2.2 `router/index.ts`

构建步骤：

1. 创建路由实例。
2. 配置默认重定向到 `/setup`。
3. 注册 `setup/board/about` 页面。
4. about 使用懒加载，减少初始包大小。

#### 4.2.3 `lib/board.ts`（前端数据核心）

构建步骤：

1. 先定义业务模型与约束常量（最小等级、最大上传数等）。
2. 封装配置默认值与 ID 生成。
3. 封装 localStorage 读写。
4. 增加严格校验与兼容恢复，防止脏数据污染页面。
5. 提供构建状态的方法（`createBoardStateFromConfig`）。

#### 4.2.4 `SetupView.vue`（配置与输入）

构建步骤：

1. 构建榜单配置表单（标题、等级文案、颜色）。
2. 增加图片输入能力：
   - 选择文件
   - 粘贴文本解析 URL
   - 拖拽文件/链接/网页内容
3. 调用后端解析接口，统一转换提示信息。
4. 维护失败图片状态与信息反馈。
5. 点击“进入榜单”时，保存配置并重置排序状态。

#### 4.2.5 `BoardView.vue`（排序与导出）

构建步骤：

1. 从配置和状态初始化榜单。
2. 通过 `vuedraggable` 绑定每个等级数组与 pool 数组。
3. 增加图片加载失败兜底。
4. 增加导出按钮与下载流程（`html2canvas`）。
5. 增加导出状态提示与失败提示。

## 5. 构建与运行流程（工程层）

### 5.1 后端

开发运行：

```bash
mvn spring-boot:run
```

编译检查：

```bash
mvn -DskipTests compile
```

打包：

```bash
mvn package
```

### 5.2 前端

安装依赖：

```bash
cd vue_atd
npm install
```

开发运行：

```bash
npm run dev
```

构建：

```bash
npm run build
```

### 5.3 联调关键点

- Vite 代理 `/api` 到 `http://localhost:8080`
- 前端无需关心后端跨域细节（由代理与后端图片代理共同解决）

## 6. 你可以重点掌握的学习路径

建议按以下顺序学习本项目：

1. 先读 `board.ts`：理解业务数据模型与状态持久化。
2. 再读 `SetupView`：理解输入来源与解析流程。
3. 再读 `ImageResolveService`：理解后端解析、安全、缓存策略。
4. 最后读 `BoardView`：理解拖拽排序与导出图像实现。

掌握这四块后，整个项目可独立维护与扩展。
