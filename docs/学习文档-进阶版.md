# ATD 学习文档（进阶版）

## 1. 架构关注点

ATD 的核心不是“页面展示”，而是“图片来源治理 + 前端状态闭环”。

系统拆分后的关注点：

1. 后端负责不可信输入治理（URL 校验、SSRF 防护、大小限制、超时）
2. 前端负责高频交互（拖拽、局部持久化、导出）
3. 两端通过简单 DTO 协议解耦（`ResolveImageRequest/Response`）

---

## 2. 后端深入

### 2.1 解析链路

`resolve(rawUrl)` 关键流程：

1. `parseAndValidate`：协议/主机校验
2. 缓存命中检查（`resolveCache`）
3. `resolveInternal`：
   - 直链后缀快速识别
   - 抓取 HTML（或直接命中 image content-type）
   - 提取候选图片并返回代理 URL

### 2.2 SSRF 防护策略

策略在 `parseAndValidate` + `isPublicHost` + `isPrivateAddress`：

- 禁止 `localhost` / `.local`
- DNS 解析后检查是否为私网、回环、链路本地、组播等
- 拒绝非 `http/https`

这保证了解析服务不会被用来探测内网资源。

### 2.3 可用性与资源控制

`fetch()` 配置：

- 连接超时、请求超时
- 最大响应体限制（HTML 与图片分别限制）
- 统一 User-Agent

收益：

- 防止慢站点拖死线程
- 防止大文件导致内存暴涨
- 为失败分类提供稳定 reason

### 2.4 解析策略质量

提图优先级：

1. `og:image` / `twitter:image`
2. `img[src|data-src|data-original]` 回退
3. 按宽高评分选最优

可继续优化方向：

- 增加 `srcset` 支持
- 识别懒加载框架自定义属性
- 引入语义打分（正文区域优先）

## 3. 前端深入

### 3.1 状态模型设计

`board.ts` 拆为两层：

1. 配置层 `BoardConfig`：标题、等级、图片清单
2. 运行层 `BoardState`：每个等级数组 + 待排序池

这种分层让“配置”和“过程状态”可独立恢复、重置、迁移。

### 3.2 拖拽状态一致性

`vuedraggable` 通过 `v-model` 修改数组，`watch(board, { deep: true })` 立刻持久化。  
核心风险点在恢复阶段，`toCompatibleState()` 做了严格校验：

- 每张图只能出现一次
- 图 ID 与 src 必须匹配配置快照
- 所有图片必须被分配到 rank 或 pool

这套校验是避免“脏状态”导致界面错乱的关键。

### 3.3 输入通道统一

`SetupView` 将多输入形态汇总：

- 文件输入 -> `addImagesFromFiles`
- 文本输入 -> `extractHttpUrls` -> `addImagesFromText`
- 拖拽输入 -> 优先文件，再解析 URI/HTML

设计重点：

- 对用户入口多样化
- 对内部处理统一化

### 3.4 导出实现与兼容

`BoardView` 导出流程：

1. 锁定导出区域（`rankTableRef`）
2. 等待图片可解码
3. `html2canvas` 局部渲染
4. 触发浏览器下载

针对 `object-fit` 兼容问题，导出时在克隆 DOM 将 `img` 替换为背景块（`background-size: cover`），保证高图比例一致。

## 4. 构建与工程化

### 4.1 前端构建链

- `vite`：开发与打包
- `vue-tsc`：模板+TS 类型检查
- `npm-run-all2`：并行执行检查和构建

`package.json` 的 `build` 实际是：

1. 先 `type-check`
2. 再 `vite build`

### 4.2 后端构建链

- Maven + `spring-boot-maven-plugin`
- Java 17
- 可通过 `mvn -DskipTests compile` 快速校验语法

---

## 5. 扩展建议（进阶实战）

1. 服务端缓存升级为 Redis，支持多实例共享缓存。
2. 图片解析做异步任务化，前端轮询结果。
3. 导出升级为服务端渲染，稳定处理跨域图片。
4. 引入监控指标：
   - 解析成功率
   - 平均解析耗时
   - 失败原因分布
5. 为 `ImageResolveService` 增加集成测试样例库（不同站点 HTML 结构）。

---

## 6. 阅读源码建议（进阶顺序）

1. `ImageResolveService`（安全+解析主流程）
2. `board.ts`（状态一致性）
3. `SetupView`（输入归一）
4. `BoardView`（导出与拖拽）

按这个顺序，你可以快速建立“系统级”理解，而不只是“页面级”理解。
