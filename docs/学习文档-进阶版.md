# ATD 学习文档（进阶版，纯前端）

## 1. 设计关注点
纯前端版本的核心在于：

1. 输入治理：只收敛到“本地文件 + 图片直链 URL”
2. 状态闭环：配置层与运行层分离，保证可恢复
3. 导出稳定性：导出前等待图片完成加载，避免空白图

## 2. 输入链路拆解
`SetupView` 处理三类输入：

- 文件：`FileReader -> dataURL`
- 文本：提取 URL -> 直链校验 -> `Image` 加载探测
- 拖拽：优先文件，回退文本/HTML 链接提取

直链校验不通过（如网页链接）直接失败提示，不再尝试抓取 HTML。

## 3. 状态模型与一致性
`board.ts` 将状态拆成：

- `BoardConfig`：相对静态配置
- `BoardState`：动态拖拽结果

读写策略：

- 先写内存，再尝试写 `localStorage`
- 读取优先内存，失败再读 `localStorage`

这样可以降低存储上限或隐私模式导致的数据丢失风险。

## 4. 导出链路
`BoardView` 导出流程：

1. 收集 `rank-table` 区域
2. 等待图片 `decode/load/error`
3. `html2canvas` 按高分辨率参数渲染
4. 生成 PNG 下载

失败图片不会阻断导出，只会在结果中保留 fallback 区块。

## 5. 可演进方向
1. 增加 URL 白名单/黑名单策略（纯前端规则层）。
2. 增加导出主题模板（边框、标题样式、尺寸比例）。
3. 增加离线存档（导入导出 JSON）。
